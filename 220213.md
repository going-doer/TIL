## 2022.02.13
### 핸즈온 머신러닝 1부
### 연습문제

1. 머신러닝을 어떻게 정의할 수 있나요?
- 데이터로부터 학습하도록 컴퓨터를 프로그래밍 하는 과학(또는 예술)
- 명시적인 프로그래밍 없이 컴퓨터가 학습하는 능력을 갖추게 하는 연구 분야

- 데이터로부터 스스로 학습하는 능력을 가진 컴퓨터를 갖추게 하는 연구 분야


2. 머신러닝이 도움을 줄 수 있는 문제 유형 4 가지를 말해보세요.
- 기존 솔루션에서 수동 조정 및 규칙이 필요한 문제
- 전통적인 방법으로는 해결하기 어려울 정도의 복잡한 문제
- 새로운 환경에 쉽게 적용하는 특성을 살린 유동적인 환경을 가진 문제
- 복잡한 문제와 대량의 데이터에서 통찰, 패턴을 찾는 것
    - data mining
 
3. 레이블된 훈련 세트란 무엇인가요?
- 지도학습에서 사용되고, 데이터에 대한 답을 레이블이라는 것으로 적어둔 훈련 세트

4. 가장 널리 사용되는 지도 학습 두 가지는 무엇인가요? 
- 분류 classification
    - 스팸 분류
- 예측 변수(predictor variable)이라고 불리는 특성(feature)을 사용하여 타깃(target) 수치를 예측
    - 주행거리, 연식, 브랜드를 사용하여 중고차의 가격을 예측하는 것.

5. 보편적인 비지도 학습 작업 네가지는 무엇인가요?
- 차원 축소(dimensionality reduction)
    - 데이터를 간소화 하는 것
    - 상관관계가 있는 여러 특성을 하나로 합치는 것, 특성 추출(feature extraction)
- 이상치 탐지(outlier detection)
    - 이상 샘플을 인식하도록 훈련하여, 새로운 샘플이 정상 데이터인지 이상치인지 판단하는 것.
    - 이상한 신용카드 거래 탐지, 제조 결함
    - 데이터 셋에서 이상한 값을 자동 제거
- 특이치 탐지(novelty detection)
    - 훈련 세트에 있는 모든 샘플과 달라 보이는 샘플을 탐지
    - 훈련 세트의 샘플은 매우 '깨끗한' 상태여야 함.
- 연관 규칙 학습(assocition rule learning)
    - 대량의 데이터에서 특성 간의 흥미로운 관계를 찾는 것.
    - 슈퍼마켓에서 바비큐 소스와 감자사면 스테이크도 삽니다.

6. 사전 정보 없는 여러 지형에서 로봇을 걸어가게 하려면 어떤 종류의 머신러닝 알고리즘을 사용할 수 있나요?
- 강화학습 알고리즘
    - 주어진 상황에서 보상을 얻기 위해 정책(policy)라고 부르는 최상의 전략을 스스로 학습

7. 고객을 여러 그룹으로 분할하려면 어떤 알고리즘을 사용해야 하나요?
- 계층 군집 알고리즘(hierachical clustering)
- Top-down 방식, 분할 군집(divise clustering)

8. 스팸 감지의 문제는 지도 학습과 비지도 학습 중 어떤 문제로 볼 수 있나요?
- 지도 학습의 분류(classification)

9. 온라인 학습 시스템이 무엇인가요?
- 데이터를 점진적으로 받아와서 이전의 데이터에 누적해서 학습하는 방식
- 데이터를 순차적으로 한 개씩 또는 미니 배치(mini-batch)라고 부르는 작은 묶음 단위로 주입해서 시스템을 훈련.
- 학습 단계가 빠르고 적은 비용
- 데이터가 도착하는 대로 학습 가능.
- 학습 후 필요하지 않을 경우 버리면 되기 때문에 공간 절약.
- 적합한 유형
    - 연속적으로 데이터를 받고 빠른 변화에 스스로 적응해야 하는 시스템
    - 컴퓨팅 자원이 한정된 경우

10. 외부 메모리 학습이 무엇인가요?
- 점진적으로 학습하는 방식으로 메모리가 작은 디바이스에서 학습을 할 때 데이터의 양을 Mini batch로 조각내서 incremental하게 학습 시키는 것.
- 컴퓨터 한 대의 메모리에 들어갈 수 없는 아주 큰 데이터 셋을 학습하는 시스템에서 온라인 학습 알고리즘을 사용하는 것
- 알고리즘이 데이터의 일부를 읽고 훈련을 반복하며, 전체 데이터가 모두 적용될 때까지 반복.

11. 예측을 하기 위해 유사도 측정에 의존하는 학습 알고리즘은 무엇인가요?
- 사례 기반 학습 알고리즘(instance-based learning)

12. 모델 파라미터와 학습 알고리즘의 하이퍼파라미터 사이에는 어떤 차이가 있나요?
- 하이퍼파라미터는 데이터셋의 양을 조정하는 것으로 최소적합 최대적합
- 모델 파라미터
    - 모델 파라미터를 조정하여 최상의 성능을 낼 수 있는 모델을 만들어 낸다.
    - 훈련(training) 시킨다 = 훈련 데이터에 가장 잘 맞는 모델의 파라미터를 찾는 과정.
    - 훈련을 하면서 모델 파라미터는 계속 변함.
- 하이퍼파라미터
    - 학습하는 동안 적용할 규제의 양(하이퍼파라미터를 감소 = 제약,규제를 줄인다.)
    - 학습 알고리즘의 파라미터
    - 훈련 전에 지정 되며, 훈련하는 동안에는 상수로 남아 있음.

13. 모델 기반 알고리즘이 찾는 것은 무엇인가요? 성공을 위해 이 알고리즘이 사용하는 가장 일반적인 전략은 무엇인가요? 예측은 어떻게 만드나요?
- 예측(predication)
- 모델을 선택하고, 해당 모델의 최적 모델 파라미터의 값을 알아내기.
    1. 데이터를 분석
    2. 모델을 선택
    3. 훈련 데이터로 모델을 훈련(학습 알고리즘이 비용 함수를 최소화 하는 모델 파라미터를 찾는 것)
    4. 새로운 데이터에 모델을 적용해 예측을 하고(추론 inference), 이 모델이 일반화되길 기대.
    
14. 머신러닝의 주요 도전 과제는 무엇인가요?
- 나쁜 데이터
    - 충분하지 않은 양의 훈련 데이터
    - 대표성 없는 훈련 데이터
        - 샘플링 잡음(sampling noise): 샘플이 작은 경우 
        - 샘플링 편향(sampling bias): 샘플이 크지만 표본 추출이 잘못 될 경우
    - 낮은 품질의 데이터
    - 관련 없는 특성
        - 특성 선택(feature selection): 특성 중 가장 유용한 특성을 선택
        - 특성 추출(feature extraction): 특성을 결합하여 더 유용한 특성을 만듦. 차원 축소 알고리즘
    - 훈련 데이터 과대적합(overfitting)
        - 모델이 훈련 데이터에만 너무 잘 맞고, 일반성은 떨어지는 것.
        - 모델의 제약을 가하는 규제(regulation)을 통해 균형을 맞춘다.
            - 하이퍼파라미터(hyperparameter)
    - 훈련 데이터 과소적합(underfitting)
        - 모델이 너무 단순해서 내재된 구조를 학습하지 못함.
        - 모델 파라미터가 많은 강력한 모델 선정
        - 학습 알고리즘에서 더 좋은 특성을 선택(특성 공학)
        - 모델의 제약을 줄임(하이퍼파라미터를 감소)
- 나쁜 알고리즘

15. 모델이 훈련 데이터에서의 성능은 좋지만 새로운 샘플에서의 일반화 성능이 나쁘다면 어떤 문제가 있는 건가요? 가능한 해결책 세 가지는 무엇인가요?
- 데이터 과대적합: 훈련세트에만 너무 모델이 편향된 상태
- 파라미터가 적은 모델 선택하거나 훈련 데이터에 있는 특성 수를 줄이거나, 모델에 제약을 걸어 단순화.
- 훈련 데이터의 잡음을 줄임. (오류 데이터 수정과 이상치 제거)
- 훈련 데이터를 많이 확보

16. 테스트 세트가 무엇이고 왜 사용해야 하나요?
- 훈련 데이터를 훈련 세트와 테스트 세트로 나누어서, 테스트 세트를 통해 모델을 테스트.
- 모델 평가, 적정한 모델로 구성되어있는지 확인하는 용도. 
- 일반화 오차(generalization error, out-of-sample error)
    - 테스트 세트에서 모델을 평가함으로 해당 오차의 추정값(estimation)을 얻는다.
    - 훈련 오차는 낮고, 일반화 오차는 높다 = 데이터 과대적합.

17. 검증 세트의 목적은 무엇인가요?
- 훈련 세트의 일부를 떼어내어 검증 세트(validation set)로 만들고, 이를 통해서 가장 높은 성능을 내는 모델을 선택.
- 검증 세트에서 선택된 모델을 전체 훈련 세트(여기에는 검증 세트가 포함)에서 훈련하여 최종 모델.
- 최종 모델을 테스트 세트로 평가하여 일반화 오류를 추정.
- 검증 세트를 여러 개해서 평균 치를 통해 최적의 모델을 선택하는 교차검증을 통해 더 좋은 모델을 선택할 수 있다.
    - 그러나 시간 또한 오래 걸린다.
